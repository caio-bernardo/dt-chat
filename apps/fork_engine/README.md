# Fork Engine

Create Digital Twins of Conversation Agents (and its users) from a stream of
messages. Can be configured to produce forks on a event-trigger system.

## Features

How this works?

First queue each touchpoint from the stream.

If under condition.

Query all messages from said conversation.

Spin up a UserBot using the same persona (must be on the metadata of the message) and put the same history.
Spin up a Modified bancobot with the same history.

Let the conversation run for n iterations.

Feed messages back to the classifier.
