# Digital Twin Chat/dt-chat

This file provide guidance to AI agents when working with code in this repository.

## Project Overview

Digital Twin Chat (dt-chat) is a framework to execute, monitor and simulate text based conversational agents in the scope of customer-service in banking domain. The framework has modules to generate conversations between two agents (user and chatbot), classify messages according to a set of predefined classes, and generate simulations from a catalyst point by analyzing the classes generated previously. The system works through a set of processes communicating via a Redis queue.

### Naming

Always refer to the labels generated by the classifier module as **touchpoints**.

## Project Setup

### Build Env

- Use `uv sync` to download all dependencies.
- Configure `.env` using `.env.example` as a template.
- Create text-embeddings using `scripts/embendder.py`.

### Commands

- `just <command> <args>`: execute a command, use `just --list` to see all available commands.

- `data/`: files consumed by scripts and packages.
  - `interacoes_json`: 4k conversations in json files, can be extracted by the `importer.py` script.
  - `personas/`: contain json files describing personas to generate conversations, can be used by `swarm.py` and by `importer.py`.
  - `RAG-Cartoes`: documents about the banking domain, used for Retrieve Augmentated Generation, used by `embendder.py`script.
  - `touchpoints`: `json` files with set of touchpoints for the `classifier`.
- `libs/`: python auxiliary libraries
  - `chatbot/`: declares a basic chatbot agent.
  - `pubsub/`: declares an interface for the redis queue and publish/subscribe model
  - `timesim`: declares data models for the timestamp simulation inside the conversations.
  - `userbot`: descriv
- `logs/`: contains logs of a simulation. **NEVER ALTER THEIR CONTENTS**.
- `scripts/`: utility scripts for pre configuring and pre-hooks in the framework flow;
  - `embendder.py`: generate text embenddings to be used by the RAGs agents, main target is `RAG-Cartoes/`. Need to be called once during project configuration, use `scripts/embendder.py --help` to see args.
  - `ensembler.py`: merge the classifications of many models and produces a single database using a voting algorithm.
  - `importer.py`: inserts the contents of a `json` file to the native format of this project.
  - `injector-by-id.py`: injects only a row of a database into a redis queue.
  - `injector.py`: injects all rows of a table into a redis queue.
  - `swarm.py`: simulate a swarm of users interacting with the assistant chatbot.

`justfile` has a collection of rules and commands to execute in this project, use `just <command> <args>` to execute a command, to see all available commands, use `just --list`.

The `docker-compose.yml` contains docker services like the Redis instace.

The `pyproject.toml` at the root includes all directories and packages created in `apps/`, `libs/` and `scripts/` in this workspace, dependencies can be added here to be accessible in `libs` and `apps`.

The `uv.lock` file is generated by `uv sync` and `uv lock`, it contains a snapshot of all dependencies used in the project. In case of conflict, delete it and run `uv lock` to create it again.

## Boundaries

- **Ask-first**
  - Large cross-package refactors
  - New dependencies
  - Destructive data or migration changes
- **Never**
  - Edit the contents of logs unless explicitly requested.
  - Commit secrets, crendentials, or tokens.
  - Edit generated files by hand when a generation workflow exists.
  - Use git operations unless explicitly requested.
